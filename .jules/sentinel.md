## 2026-02-27 - Insecure SSL Contexts and SSRF in Utility Scripts
**Vulnerability:** Multiple Python utility scripts (`preflight-check.py`, `batch-preflight.py`, `clean-queue.py`, `validate-queue-urls.py`) explicitly disabled SSL certificate verification (`ssl.CERT_NONE`) and lacked protocol validation for URLs passed to `urllib.request.urlopen`.
**Learning:** Developers often disable SSL verification globally in small scripts to bypass local environment issues or self-signed certificate errors during development, which then leaks into production-critical tools. Additionally, `urllib.request` handles schemes like `file://` by default, leading to Local File Disclosure (SSRF) if the protocol is not strictly validated.
**Prevention:** Always use `ssl.create_default_context()` without overrides. Enforce strict `^https?://` protocol validation for any user-supplied or queue-sourced URLs before network execution. Use `os.path.expanduser` for file paths to avoid leaking local usernames in source code.
